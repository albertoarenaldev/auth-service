## ✨ Funcionalidades completadas

### Autenticación core (Fases 1-4)
- Register con verificación de email (token opaco SHA-256, envío asíncrono)
- Login con BCrypt strength 12 y anti-enumeration
- Refresh token rotation con detección de reuso + revocación de familia
- Logout idempotente
- JWT híbrido: access stateless (HS256, 15 min) + refresh opaco en BD (7 días)

### Recuperación de cuenta (Fase 5)
- Flujo forgot-password / reset-password con token opaco SHA-256
- Anti-user-enumeration: `/forgot-password` siempre 202
- Política NIST SP 800-63B vía zxcvbn4j (score ≥ 3)
- Revocación de sesiones post-reset (OWASP ASVS V3.5)

### Perfil de usuario (Fase 6)
- `GET/PUT /api/v1/users/me` con JWT
- `POST /api/v1/users/me/password` con verificación de contraseña actual
- Revocación de todas las sesiones al cambiar contraseña

### Seguridad hardening (Fases 7-8)
- Rate limiting con Bucket4j en `/login` (5/min) y `/forgot-password` (3/5min)
- Respuesta 429 + header `Retry-After`
- Audit log append-only con 11 tipos de eventos
- IP capturada vía `RequestContextHolder` en cada evento

### Social login (Fase 9)
- OAuth2/OIDC con Google y GitHub
- Creación/vinculación automática de cuentas locales
- Redirect al frontend SPA con JWT + refresh token

### Observabilidad (bonus)
- Micrometer + Prometheus con 8 contadores de negocio
- Endpoint `/actuator/prometheus` protegido
- Métricas JVM, HikariCP, Tomcat vía Actuator autoconfig

### Mantenimiento (bonus)
- `@Scheduled` de limpieza horaria de tokens expirados (refresh, password-reset, email-verification)
- Flyway V1-V4 con migraciones idempotentes
